# Skills – Contentstack Kickstart: Next.js Middleware

Source of truth for detailed guidance. Read [`AGENTS.md`](../AGENTS.md) first, then open the skill that matches your task.

## When to use which skill

| Skill folder | Use when |
| --- | --- |
| `dev-workflow` | Installing, running, linting, env files, stack seeding, or understanding CI workflows |
| `contentstack-integration` | Changing the middleware API route, live preview, or Contentstack fetch flow |
| `testing` | Checking quality without a dedicated test suite (lint + manual verification) |
| `code-review` | Reviewing or preparing a pull request for this repo |
| `framework` | Next.js App Router layout, `next.config`, TypeScript paths, or Tailwind/PostCSS |

Each folder contains `SKILL.md` with YAML frontmatter (`name`, `description`).
